## 1. 项目概述

### 1.1 项目背景
校园内存在大量闲置物品（教材、电子产品、生活用品等），缺乏便捷的交易渠道。学生毕业季更是有集中处理闲置物品的需求。本平台旨在为校园用户提供一个安全、便捷的二手物品交易平台。

### 1.2 产品定位
- **产品类型：** PC端Web应用
- **核心价值：** 连接校园内的买卖双方，促进闲置物品流转
- **差异化：** 针对校园场景优化，支持实名认证和信用体系

### 1.3 成功指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 注册用户数 | 500+ | 上线后一个月内 |
| 日活用户数 | 100+ | 上线后一个月内 |
| 商品发布数 | 1000+ | 上线后一个月内 |
| 成交率 | >15% | 发布商品中成功交易的比例 |

## 2. 目标用户

### 2.1 用户画像

#### 主要用户：在校学生
- **特征：** 本科生、研究生，年龄18-28岁
- **需求：** 购买便宜的二手教材、电子产品；出售闲置物品回血
- **痛点：** 缺乏可信的校内交易渠道，担心被骗

#### 次要用户：毕业生
- **特征：** 即将离校，有大量物品需要处理
- **需求：** 快速出售闲置物品，减少搬家负担
- **痛点：** 时间紧迫，需要高效处理大量物品

### 2.2 使用场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 日常出售 | 学生有闲置物品，希望出售变现 | 中 |
| 求购需求 | 需要特定二手物品（如旧版教材） | 中 |
| 毕业清仓 | 毕业生集中出售大量物品 | 季节性高峰 |
| 浏览发现 | 无明确需求，浏览看看有没有好物 | 高 |

## 4. 功能需求详述

### 4.1 用户模块

#### 4.1.1 注册/登录

**功能描述：** 用户通过邮箱和密码登录系统

**需求细节：**
- 登录方式：邮箱 + 密码
- 首次使用需注册，填写邮箱、昵称、设置密码、选择所属校区
- 邮箱作为唯一账号，注册后不可修改
- 支持"记住我"功能
- 支持密码找回（通过注册邮箱接收验证码重置）

**用户流程：**
```
访问平台 → 点击登录 → 输入邮箱密码 → 验证通过 → 进入首页
    ↓ (未注册)
点击注册 → 填写邮箱、昵称、密码、校区 → 邮箱唯一性校验 → 注册成功 → 自动登录
```

**业务规则：**
- 邮箱格式：合法邮箱地址，必须唯一
- 昵称要求：2-20位，可包含中文、字母、数字
- 密码要求：8-20位，包含字母和数字
- 同一邮箱只能注册一次
- 连续登录失败5次，账号锁定30分钟
- 校区选择：预设校区列表（如东校区、西校区、南校区等）

#### 4.1.2 实名认证（可选）

**功能描述：** 验证用户真实身份，增加平台信任度（用户自愿申请）

**需求细节：**
- 认证方式：上传学生证照片/学信网截图
- 认证状态：未认证、审核中、已认证、认证失败
- 所有用户均可发布商品，实名认证为可选的信任标识
- 已认证用户显示"已认证"标识，增加买家信任度
- 认证信息仅管理员可见，不公开展示

**业务规则：**
- 认证审核时间：24小时内
- 认证失败可重新提交
- 每个账号最多提交3次认证
- 已认证用户在商品详情页显示认证标识

#### 4.1.3 个人主页

**功能描述：** 展示用户基本信息和交易数据

**展示内容：**
- 头像、昵称
- 认证状态标识（已认证显示认证图标）
- 信用评分（好评率）
- 发布商品数、成交数
- 注册时间
- 收到的评价（最近10条）

**信用评分规则：**
- 新用户默认显示"暂无评分"
- 首次交易完成后开始计算好评率
- 好评率 = 好评数 / 总评价数 × 100%
- 展示格式：好评率95%（19/20）
- 5星和4星视为好评，3星及以下视为差评

**页面元素：**
```
┌─────────────────────────────────────┐
│  [头像]  昵称                        │
│          [已认证] 好评率：95%         │
├─────────────────────────────────────┤
│  发布：12    成交：8    好评率：95%   │
├─────────────────────────────────────┤
│  最近评价：                          │
│  ⭐⭐⭐⭐⭐ "卖家很nice，物品和描述一致"  │
│  ⭐⭐⭐⭐ "交易顺利"                   │
└─────────────────────────────────────┘
```

#### 4.1.4 我的发布

**功能描述：** 管理已发布的商品

**功能列表：**
- 查看所有已发布商品
- 商品状态筛选：在售、已售、已下架、审核中、草稿
- 编辑商品信息
- 上架/下架操作
- 标记为已售
- 删除商品

**业务规则：**
- 已售商品保留记录，但不在搜索结果中显示
- 下架商品可重新上架
- 删除操作需二次确认

#### 4.1.5 交易记录

**功能描述：** 查看历史交易记录

**展示内容：**
- 作为卖家的交易记录
- 作为买家的交易记录
- 交易状态：进行中、已完成、已取消
- 交易详情：商品信息、交易时间、交易对象、取消原因（如有）

**筛选功能：**
- 按状态筛选：进行中、已完成、已取消
- 按角色筛选：我是卖家、我是买家

### 4.2 商品模块

#### 4.2.1 发布商品

**功能描述：** 卖家发布二手商品信息

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品标题 | 文本 | 是 | 1-50字，简明描述商品 |
| 商品分类 | 下拉选择 | 是 | 书籍教材/电子数码/生活用品/服饰美妆 |
| 商品描述 | 富文本 | 是 | 1-1000字，详细描述商品情况 |
| 商品图片 | 图片上传 | 否 | 最多9张，第一张为封面 |
| 转让价格 | 数字 | 是 | 单位：元，支持小数 |
| 原价 | 数字 | 否 | 帮助买家判断折扣 |
| 成色 | 下拉选择 | 是 | 全新/几乎全新/轻微使用痕迹/明显使用痕迹 |
| 交易地点 | 文本 | 是 | 建议填写校园内具体位置 |
| 发布方式 | 单选 | 是 | 立即发布/存为草稿 |

**图片要求：**
- 支持格式：JPG、PNG、WEBP
- 单张大小限制：5MB
- 建议尺寸：800x800以上
- 第一张自动作为封面图

**用户流程：**
```
点击"发布" → 填写商品信息 → 上传图片 → 选择发布方式
                                          ↓
                              ┌───────────┴───────────┐
                              ↓                       ↓
                         立即发布                  存为草稿
                              ↓                       ↓
                         商品上架                  保存草稿
                              ↓
                         被举报 → 进入审核队列
                              ↓
                         管理员审查 → 通过/下架
```

**业务规则：**
- 每个用户最多同时发布20件商品
- 草稿保存后30天内未发布自动删除
- 发布后默认上架，无需人工审核
- 用户可举报违规商品，被举报商品进入审核队列
- 管理员可审查被举报商品，决定通过或下架
- 违禁商品（如管制刀具、违禁药品）禁止发布
- 支持编辑已发布的商品（通过 `/product/publish?edit={id}` 访问）

### 4.3 交易模块

#### 4.3.1 私信沟通

**功能描述：** 买卖双方站内沟通

**功能列表：**
- 发起私信（从商品详情页）
- 文字消息
- 图片消息（单张限制5MB）
- 消息记录查看
- 未读消息提醒

**页面布局：**
```
┌─────────────────────────────────────────────────────────────┐
│  消息列表                    │        聊天窗口               │
├─────────────────────────────┤───────────────────────────────┤
│  [头像] 小明                 │  [商品卡片]                   │
│  最近消息预览...            │  iPad 2024款 ¥128            │
│  2分钟前                    │───────────────────────────────┤
│                             │  买家：你好，还在吗？          │
│  [头像] 小红                 │  卖家：在的                   │
│  最近消息预览...            │  买家：能便宜点吗？100行吗     │
│  1小时前                    │  卖家：最低110                │
│                             │───────────────────────────────┤
│                             │  [输入框]              [发送]  │
└─────────────────────────────┴───────────────────────────────┘
```

**业务规则：**
- 每个对话关联一件商品
- 消息保存90天
- 举报功能集成在聊天界面
- 已售商品仍可查看历史聊天记录

#### 4.3.2 议价协商

**功能描述：** 买家出价，卖家还价

**交互流程：**
```
买家浏览商品 → 觉得价格高 → 点击"出价"
                              ↓
                     输入期望价格 → 发送出价
                              ↓
                     卖家收到通知 → 接受/拒绝/还价
                              ↓
                     ┌─────────┼─────────┐
                     ↓         ↓         ↓
                   接受      拒绝       还价
                     ↓         ↓         ↓
                双方确认   买家可重新   买家考虑
                     ↓       出价       接受/拒绝
                线下交易
```

**功能细节：**
- 买家可输入期望价格并留言
- 卖家可接受、拒绝或还价
- 支持多轮议价
- 议价记录在聊天中显示

**业务规则：**
- 出价价格不能为负数
- 议价记录永久保存
- 卖家接受买家出价后，买家需手动点击"确认购买"创建交易记录
- 买家确认购买后，进入交易确认流程

#### 4.3.3 交易确认

**功能描述：** 线下交易完成后确认

**流程：**
```
双方协商一致 → 买家点击"确认购买" → 创建交易记录
                                          ↓
                                 约定线下交易时间和地点
                                          ↓
                                 线下见面交易
                                          ↓
                              卖家在聊天中标记"已交付"
                                          ↓
                              买家收到通知，确认收货
                                          ↓
                       ┌──────────────────┴──────────────────┐
                       ↓                                     ↓
                  买家确认收货                            超时未确认
                       ↓                                     ↓
                  交易完成                                  自动完成
                       ↓                                     ↓
                  进入评价                                  进入评价
```

**功能细节：**
- 卖家标记"已交付"后，买家收到通知
- 买家需在卖家标记后7天内确认收货
- 超时未确认自动完成交易
- 交易完成后商品状态变为"已售"

**业务规则：**
- 交易确认后不可撤销
- 卖家标记"已交付"后，买家有7天确认期
- 双方都确认后进入评价环节

#### 4.3.4 信用评价

**功能描述：** 交易完成后双方互评

**评价内容：**
- 评分：1-5星
- 评价文字（可选）
- 评价标签（可选）：描述准确、沟通顺畅、交易守时等

**展示位置：**
- 商品详情页（卖家评价）
- 用户个人主页（所有评价）
- 交易记录页

**业务规则：**
- 交易完成后7天内可评价
- 超时未评默认好评
- 评价不可修改，可追加
- 恶意评价可举报

### 4.4 系统模块

#### 4.4.1 消息通知

**功能描述：** 系统消息和互动通知

**通知类型：**

| 类型 | 触发场景 | 展示方式 |
|------|----------|----------|
| 私信通知 | 收到新私信 | 站内信+红点 |
| 互动通知 | 商品被收藏、收到出价 | 站内信+红点 |
| 系统通知 | 认证结果、审核结果、平台公告 | 站内信+红点 |
| 交易通知 | 交易确认、评价提醒 | 站内信+红点 |

**通知中心：**
```
┌─────────────────────────────────────────────────────────────┐
│  通知中心                                    [全部已读]      │
├─────────────────────────────────────────────────────────────┤
│  [私信] 小明：你好，iPad还在吗？           2分钟前   [红点]  │
│  [互动] 用户收藏了你的商品"iPad 2024"      1小时前          │
│  [系统] 你的实名认证已通过                  3小时前          │
│  [交易] 交易已完成，请对买家进行评价        1天前           │
└─────────────────────────────────────────────────────────────┘
```

**业务规则：**
- 通知保留30天
- 未读通知显示红点
- 支持全部已读操作

#### 4.4.2 审核中心

**功能描述：** 统一管理实名认证审核和商品举报审查

**审核类型：**

| 类型 | 触发条件 | 审核内容 |
|------|----------|----------|
| 实名认证审核 | 用户提交认证申请 | 学生证照片、学信网截图 |
| 商品举报审查 | 用户举报违规商品 | 商品信息、图片是否违规 |
| 用户举报审查 | 用户举报违规用户 | 用户行为是否违规 |

**审核流程：**
```
实名认证：用户提交 → 进入审核队列 → 管理员审核 → 通过/拒绝
商品举报：用户举报 → 进入审核队列 → 管理员审查 → 通过(保留)/下架
```

**审核标准：**
- 实名认证：证件清晰、信息真实
- 商品举报：是否包含违禁内容、虚假信息、广告引流

**业务规则：**
- 实名认证审核时间：24小时内
- 商品举报审查时间：24小时内
- 审核状态实时通知用户
- 拒绝需说明原因

#### 4.4.3 举报机制

**功能描述：** 用户举报违规内容

**举报类型：**
- 虚假商品
- 违禁商品
- 欺诈行为
- 不当言论
- 其他

**举报流程：**
```
用户发现违规 → 点击举报 → 选择类型 → 填写说明 → 提交
                                                      ↓
                                             进入审核队列
                                                      ↓
                                            管理员处理举报
                                                      ↓
                                       ┌──────────────┴──────────────┐
                                       ↓                             ↓
                                   举报成立                     举报不成立
                                       ↓                             ↓
                                  处罚违规用户                  通知举报人
```

**业务规则：**
- 举报后24小时内处理
- 举报人信息保密
- 恶意举报将被处罚

#### 4.4.4 违规处理

**功能描述：** 处理违规用户和内容

**处罚措施：**

| 违规程度 | 处罚措施 | 说明 |
|----------|----------|------|
| 轻微 | 警告 | 首次轻微违规 |
| 一般 | 限制发布7天 | 多次轻微或一般违规 |
| 严重 | 封禁账号30天 | 严重违规 |
| 特别严重 | 永久封禁 | 欺诈、违法等 |

**业务规则：**
- 处罚前需核实证据
- 用户可申诉
- 申诉结果48小时内反馈

## 6. 边缘场景处理

### 6.1 已售商品处理

**场景：** 商品交易完成后如何处理

**处理方式：**
- 卖家可手动标记为"已售"
- 双方确认交易完成后自动标记为"已售"
- 已售商品：
  - 显示"已售"标识
  - 不再出现在搜索结果中
  - 保留商品详情页（可被原买家访问）
  - 保留聊天记录

### 6.2 取消发布

**场景：** 卖家不想卖了

**处理方式：**
- 卖家可随时下架商品
- 下架操作：
  - 立即生效
  - 不在搜索结果中显示
  - 未读私信不受影响
- 重新上架：
  - 无需重新审核
  - 保留原有浏览量、收藏数

### 6.3 交易取消

**场景：** 交易过程中需要取消

**处理方式：**
- 交易取消需对方同意
- 流程：
  1. 发起方在交易记录中点击"申请取消"
  2. 对方收到通知，可同意或拒绝
  3. 双方同意后，交易取消
- 取消后：
  - 商品自动重新上架
  - 交易记录标记为"已取消"
  - 取消原因记录在交易详情中
- 对信用影响：
  - 正常取消不影响信用
  - 频繁取消（如30天内取消3次以上）将限制发布功能7天

### 6.4 账号注销

**场景：** 用户想注销账号

**处理方式：**
- 注销前需确认：
  - 所有在售商品自动下架
  - 进行中的交易需完成或取消
  - 账号数据保留30天后删除
- 注销后：
  - 无法登录
  - 商品全部下架
  - 聊天记录保留但无法发送新消息

### 6.5 系统异常

**场景：** 网络异常、服务器错误

**处理方式：**
- 网络异常：提示用户检查网络，支持重试
- 服务器错误：显示友好错误页面，记录日志
- 数据不一致：后台定时任务检测并修复
